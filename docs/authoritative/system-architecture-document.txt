System Architecture Document for a Privacy-First Edge CV Ad Platform on Jetson

Source file: System Architecture Document for a Privacy-First Edge CV Ad Platform on Jetson.pdf
Status: Authoritative architecture baseline in current source set.

Role in repository:
- Defines architecture boundaries, trust boundaries, data-retention boundaries, deployment mapping, failure modes, security/privacy controls, and validation framing.
- Establishes that privacy constraints must be enforced as system properties, not only policy statements.

Key baseline points:
- Pixels may exist only transiently in RAM/GPU memory during processing.
- No image/video frame persistence and no pixel egress from device.
- Interface boundaries and deployment views should enable independent implementation and integration.
- Playback, privacy, and operational recoverability are core architecture concerns.
